Thai PBS Video CMS — Video Upload SDK

Upload videos into Thai PBS Video CMS from your own app — one class, no dependencies.

The video file lives on the media service; Thai PBS Video CMS stores the metadata and a reference to it. CmsTwoSdk does both in one flow: upload the file, register a media-video, then create the video record — and gives you callbacks + a status you can poll.

Installation

Copy sdk.js into your project and import it — no npm install, no build step (the ByteArk upload library is loaded from a CDN inside the file).

import { CmsTwoSdk } from './sdk.js';

Quick start

Four steps from a picked file to a video in Thai PBS Video CMS.

1 Initialize the SDK

Create one upload manager with your credentials and callbacks — reuse it for every upload.

const uploadManager = new CmsTwoSdk({
  byteark: { formId, formSecret, projectKey },
  cms: {
    baseUrl:  '<cms base url>',
    accessId: '<accessId>',   // public — identifies your key
    secret:   '<secret>',     // signs appTokens locally; never sent over the wire
  },                          // no teamId — derived from your key
  onUploadProgress: (job, progress) => console.log(progress.percent + '%'),
  onUploadCompleted: (job) => console.log('created:', job.video.id),
});

2 Add the file(s) to the queue

Queue a File (from an <input type="file">), optionally with metadata.

uploadManager.addUploadJobs([{ file, title: 'My video', description: 'demo' }]);

3 Start uploading

Runs the queue and creates the Thai PBS Video CMS records. Resolves when every job is done.

const [job] = await uploadManager.start();
console.log(job.video.id);   // the Thai PBS Video CMS video id

4 Wait until it's playable

The file keeps processing after upload — poll until completed, then use embeddedUrl.

let video = await uploadManager.getVideoById(job.video.id);
while (video.mediaVideo.mediaVideoStatus !== 'completed') {
  await new Promise((r) => setTimeout(r, 5000));
  video = await uploadManager.getVideoById(job.video.id);
}
player.src = video.mediaVideo.embeddedUrl;

Uploading a single file? upload(file) does steps 2–4 in one call and returns an awaitable handle.

API reference

new CmsTwoSdk(options)

Create an upload manager. Pass your credentials once; reuse it for many uploads. All callbacks are optional.

OptionTypeDescription
bytearkobject{ formId, formSecret, projectKey } — upload form credentials.
cmsobject{ baseUrl, accessId, secret, log? }. accessId + secret are your key (from the Thai PBS team); the SDK signs an appToken with secret locally and exchanges it — the raw secret never crosses the network. No teamId — it's derived from the key. log is an optional (msg) => void.
onUploadProgressfunction(job, progress) — a job's upload progressed. progress = { percent }.
onUploadCompletedfunction(job) — a job's Thai PBS Video CMS records now exist (job.media, job.video).
onUploadFailedfunction(job, error) — a job failed; the queue continues with the next one.
onVideosCreatedfunction(videoIds) — fired once after start() drains the queue, with every created Thai PBS Video CMS video id.
onStatusfunction(job, phase) — phase is 'uploading' | 'creating-media' | 'creating-video'.
 
const uploadManager = new CmsTwoSdk({
  byteark: { formId, formSecret, projectKey },
  cms: {
    baseUrl:  '<cms base url>',
    accessId: '<accessId>',   // public
    secret:   '<secret>',     // signs appTokens locally; never sent
  },                          // no teamId — derived from your key
  onUploadProgress: (job, progress) => console.log(job.name, progress.percent + '%'),
  onUploadCompleted: (job) => console.log('created:', job.video.id),
  onUploadFailed: (job, error) => console.error(job.name, error),
  onVideosCreated: (videoIds) => console.log('all done:', videoIds),
});

addUploadJobs(files) returns Job[]

Queue one or more files for upload. Nothing uploads until you call start().

ParamTypeDescription
filesFileList | File[] | object[]Either raw files, or { file, title?, description?, programId? } objects to attach metadata per file. A bare File uses its filename as the title.
 
uploadManager.addUploadJobs(fileList);                       // from an <input type="file">
uploadManager.addUploadJobs([{ file, title: 'My video', description: 'demo', programId: '<id>' }]);

start() async → Job[]

Process every pending job in the queue, one at a time. Resolves with the full job list when all jobs are completed or failed. It never rejects — a failed job fires onUploadFailed and the queue moves on. Calling it while already running returns the in-flight promise.

const jobs = await uploadManager.start();
const ok = jobs.filter((j) => j.status === 'completed');

getJobQueue() returns Job[]

The current queue, including finished jobs. Useful for rendering per-file progress — read job.status and job.progress.percent.

for (const job of uploadManager.getJobQueue()) {
  console.log(job.name, job.status, job.progress.percent + '%');
}

upload(file, metadata?) returns handle

Single-file shortcut: queues the file and starts immediately, returning an awaitable handle so you can track just that one upload without wiring constructor callbacks.

ParamTypeDescription
fileFileA File from an <input type="file"> or drag-and-drop.
metadataobjectOptional { title?, description?, programId? }.

The handle has:

MemberDescription
.onProgress(fn)Register a (percent) => void callback. Chainable.
.onStatus(fn)Register a (phase) => void callback. Chainable.
await handleResolves to { media, video } once the records exist.
.whenReady({ intervalMs })Polls until the video is playable; resolves with the Thai PBS Video CMS video (embeddedUrl under video.mediaVideo). Default interval 5000ms.
 
const myVideo = uploadManager.upload(file, { title: 'My video' });
myVideo.onProgress((pct) => console.log(pct + '%'));

const { video } = await myVideo;           // Thai PBS Video CMS video record created
const ready = await myVideo.whenReady();   // playable video (from Thai PBS Video CMS)
player.src = ready.mediaVideo.embeddedUrl;

getVideoById(videoId) async → Video

Fetch a video by its id (GET /api/v1/videos/:videoId) — use job.video.id from the upload. Returns the full video record. The transcoding status and player URL live under mediaVideo; poll until mediaVideo.mediaVideoStatus === 'completed', then play mediaVideo.embeddedUrl.

Note: the top-level mediaStatus is the publish status (e.g. draft), not the transcoding status.

listPrograms() async → Program[]

List the team's programs, for linking a video to one via programId. Each item has id, title, and slugCode.

const programs = await uploadManager.listPrograms();

The Job object

Every queued file becomes a job. Callbacks receive it, and getJobQueue() returns them.

FieldTypeDescription
file / nameFile / stringThe source file and its filename.
title / description / programIdstringMetadata for the video record.
statusstring'pending' → 'uploading' → 'creating-media' → 'creating-video' → 'completed' (or 'failed').
progressobject{ percent } — upload progress.
media / videoobjectThe created Thai PBS Video CMS records (set after each step).

Low-level functions

The class is built on these named exports. Use them directly if you need finer control. Each CMS call takes a context from makeCmsContext.

FunctionDescription
uploadVideo(options)One-shot: upload + create media + create video. Returns { media, video }. Same options as the constructor plus file, title, description, programId, onProgress, onStatus.
makeCmsContext({ baseUrl, accessId, secret, log? })Build the cms context (base URL + credentials + a token cache) the calls below need. The context signs/exchanges tokens on demand and caches the accessToken + teamId.
uploadToByteArkStream(config, file, { title }, onProgress)Upload the file only; resolves with the raw ByteArk key. You only need this if you're wiring the steps yourself — createMediaVideo consumes it internally.
createMediaVideo(cms, { videoKey, file })Register the media-video (POST /media/files, sends the token in the x-upload-token header). teamId is filled from the key. Returns the media.
createVideo(cms, { media, title, tagline, programId })Create the video record (POST /videos, x-upload-token header), linked to the media. Returns the video.
getVideoById(cms, videoId)Fetch a video (GET /videos/:videoId, public — no token); status + embeddedUrl are under video.mediaVideo.
listPrograms(cms)List the key's team's programs (team derived from the key).
 
import { uploadVideo, makeCmsContext, getVideoById } from './sdk.js';

const cmsCreds = {
  baseUrl:  '<cms base url>',
  accessId: '<accessId>',
  secret:   '<secret>',   // signs appTokens locally; never sent
};

const { video } = await uploadVideo({
  file, title: 'My video',   // no teamId — derived from the key
  byteark: { formId, formSecret, projectKey },
  cms: cmsCreds,
  onProgress: (pct) => console.log(pct + '%'),
});

const cms = makeCmsContext(cmsCreds);
const fresh = await getVideoById(cms, video.id);
